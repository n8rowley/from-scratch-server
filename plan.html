Phase 0: Requirements Specifications

The problem

  • The result of this project is a group of interconnected html pages with several types of content

The solution

  • What I Know: page style will be assisted through theuse of id's and classes
  • What I'm not sure about: I'm not sure how to implement a relative url link. Also, I don't know how to implement zebra stripes with css

Phase 1: System Analysis

Data

  • All blog page files will be in a common folder
  • links between these files will be navigated via relative URL's
  • External webpages are links to MDN's website, accomplished via absolute URL's

Output

  • HTML pages that can be browsed in a local file system.
    • Every page connects to every other page via a navigation bar. Links can be followed without knowing anything about the path

Phase 2: Design

  • Each <div> element that will have a similar background color or properties to others on the same page will be managed with a common class. Examples include:
    • blog-post
    • nav-bar
    • cheatsheet
    • bio
  • Each <div> element that requires a unique background color or other properties will be managed by a unique id. Examples include:
    • phase(0-6)
    • element
    • entity
    • selector
    • property
  • Each blog element will be contained in a <div> tag with a corresponding class attribute to assist with consistent formatting (font, size, etc) accross pages and sections
  • Picture gallery will be implemented using a flex box for horizontal display
  • Possible uses of less-obvious tags:
    • div: basically everything
    • p: blocks of content text
    • code: representing techtip entries as they would look in the source code
    • ul: listing out blocks of info
    • ol: describing the steps in executing tests
  • Possible uses of less-obvious selectors
    • Font-style: creating an emphasized byline on a blog post
    • Margin: creating white space between different divs
    • Padding: aligning text as desired

Phase 4: Testing & Debugging

Working links test

  • Procedure: For each webpage,
    1. Scroll to the top of the page where the navigation bar is visible.
    2. Click on the first link
    3. Confirm expected result
    4. Return to the original page and repeat steps 1-3 for each link
    5. Navigate to another webpage and repeat steps 1-4 for every page in the project
  • Expected Result: The browser is redirected to the page that corresponds to the link text without any other input

Valid HTML test

  • Procedure: For each wepage, upload the source file to W3C's HTML validator tool
    • https://validator.w3.org/nu/#file
  • Expected Result: The tool reports no warnings or errors

Valid CSS test

  • Procedure: For each wepage, upload the source file to W3C's CSS validator tool
    • https://validator.w3.org/nu/#file
  • Expected Result: The tool reports no warnings or errors
