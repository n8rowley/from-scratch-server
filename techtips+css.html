HTML elements

<div>
  • Purpose: This tag provides no inherent formatting to the contents. Instead, it acts as a container for seperating out content to be selected and styled with CSS
  • Interesting note: The align attribute is depricated. To align div elements on a page, use a css flex box or css grid
  • Interesting note: div tags should only be used when other pre-defined tags are not applicable
  • Default Display: Block
<code>
  • Purpose: format the contents to signify that it is a segment of computer code. The default behavior is to display the content in the defualt monospace font.
  • Default Display: In-line
  • Interesting Note: code is meant for single lines of code, in order to show a block of code including the white space wrap the block in the pre tag.
<ol>
  • Purpose: A container for a list of items in order. The default marker is a number for each item
  • Interesting Note: ol and ul elements can be nested as deeply as desired
  • Default Display: Block
<li>
  • Purpose: Represents a single item in a list. li elements must be the child of a ol, ul, or menu element.
  • Default Disply: Block

HTML Characters

&
  • Literal Interpretation: Indicates the the following text will represent an HTML entity.
  • Entity: &amp;
<
  • Literal Interpretation: Indicates the start of an HTML tag
  • Entity: &lt;
"
  • Literal Interpretation: Indicates the beginning and end of a value given to an attribute
  • Entity: &quot;

CSS Selectors

elementname
  • HTML Feature Selected: all tags of type <elementname>
  • Assignment: N/A. When an element is created, it can automatically be selected by its tag name
.class
  • HTML Feature Selected: all elements whose class attribute matches the selected class
  • Assignment: Add the attribute "class" inside the start tag of an element. Assign its value by placing the desired class name in quotes beside the class attribute followed by an = sign
  • Example: Assign the class "foot note" to a div
    • HTML assignment: <div class="foot-note">(content)</div>
    • CSS selection: .foot-note
    • Note: class names are seperated by white space. Class names, therefore, must be a single string of characters.
#id
  • HTML Feature Selected: the html element whose id attribute matches the selected id
  • Assignment: Add the attribute "id" inside the start tag of an element. Assign its value by placing the desired id in quotes beside the id attribute followed by an = sign
  • Example: Assign the id "home" to a div
    • HTML Assignment: <div id="home">(content)</div>
    • CSS Selection: #home
    • Note: id's are intended to refer to a single html element in a document. The language takes no measures to prevent multiple assignment, but the practice is discouraged. For multiple assignments, use the class attribute.
:nth-child
  • HTML Feature Selected: elements based on their position among a group of siblings
  • Assignment: N/A. The nth-child selector is a pseudo-selector. It selects elements based on the structure of the html file that it applies to.
  • Example: Select every-other liin a list starting with the first
    • CSS Selection: li:nth-child(odd)

CSS Properties

background-color
  • Purpose: change the background color of an element
  • Possible values: a color specified in one of a number of schemes. Some schemes include:
    • rgb
    • hsl
    • hex-color
    • named-color
    • and many others
margin
  • Purpose: Shorthand property for setting a margin on all four sides
  • Possible Values: 1-4 values representing the desired amount of margin for one or more sides of the element
  • Note: Margin amounts can be specified with a length, a percent, or using auto.
display
  • Purpose: Specifies whether an element is rendered in-line or as a block. Additionally, specifies the layout used for children including flow layout, grid or flex.
  • Possible Values: "in-line", "block", "flow", "flex", "grid", some combination of these and/or several others
